## 3. Firebase Secret Manager (비밀 키 보관소)

> Firebase 콘솔에서 직접 확인 불가. Google Cloud Console에서 관리.  
> **Google Cloud Console** → https://console.cloud.google.com → 프로젝트: mari-viet → Secret Manager

| 시크릿 이름 | 용도 | 비고 |
|------------|------|------|
| `OPENAI_API_KEY` | DALL-E 3 AI 이미지 생성 | OpenAI 사이트에서 발급 |
| `ADMIN_SECRET` | 최고관리자 전용 기능 잠금 | 직접 설정한 비밀번호 |
| `GMAIL_USER` | 이메일 발송용 Gmail 주소 | 예: yourmail@gmail.com |
| `GMAIL_APP_PASS` | Gmail 앱 비밀번호 | Gmail 2단계 인증 후 발급 |

### Gmail 앱 비밀번호 발급 방법 (나중에 필요할 때)
1. Gmail 로그인 → Google 계정 → 보안
2. "2단계 인증" 켜기
3. "앱 비밀번호" 생성 → 앱: 메일, 기기: Windows → 16자리 비밀번호 복사
4. Firebase Secret Manager에서 `GMAIL_APP_PASS` 업데이트

## 4. Cloud Functions (서버 함수)

> 파일 위치: `functions/index.js`  
> 런타임: Node.js 20, 리전: us-central1 (기본값)

| 함수명 | 역할 | 접근 권한 |
|--------|------|----------|
| `generateImage` | DALL-E 3로 AI 이미지 생성 후 Storage 저장 | ADMIN_SECRET 필요 |
| `getAiImages` | 생성된 AI 이미지 목록 조회 | ADMIN_SECRET 필요 |
| `sendVerificationEmail` | 회원가입 시 이메일 OTP 발송 | 누구나 호출 가능 |
| `verifyEmailOtp` | OTP 코드 확인 | 누구나 호출 가능 |

### 이메일 OTP 동작 방식
- 6자리 숫자 코드 → Firestore `email_otps` 컬렉션에 저장
- 유효시간: **10분**
- 최대 시도: **5회** 초과 시 차단
- 성공 시 Firestore에서 자동 삭제
